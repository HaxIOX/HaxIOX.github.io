完整卸载nezha-agent
2025-10-06 22:51:28

背景

由于nezha面板的服务器后套删除后，一段时间内还会生成新的，甚至是出现两条。初步怀疑是因为同时安装多个面板造成的，比如当这一条数据还没有出现在面板上的时候，复制的安装命令可能还是同一个。

杀进程卸载agent

第一步：停止正在运行的 Agent 进程

这是最关键的一步。Agent 可能以多种方式在后台运行，你需要根据当初的安装方式找到并停止它。请登录到正确的服务器（用面板上显示的IP地址登录），然后依次尝试以下方法：

方法一：检查直接运行的进程 (最通用)

这是查找任何残留进程的最终方法。

  1. 查找进程ID (PID):

    ps aux | grep nezha-agent

  2. 分析结果：

    • 如果你看到两行结果，其中一行不包含 grep，那么你就找到了真正的 Agent 进程。记下它的 PID（通常是第二列的数字）。

      # 示例输出 (找到了!)
root      1234  0.1  0.5 ... /opt/nezha/agent/nezha-agent -s ...
root      5678  0.0  0.1 ... grep nezha-agent

    • 如果你只看到一行，且该行包含 grep nezha-agent，那么说明没有 Agent 进程在运行，你可以直接跳到第二步。

  3. 杀死进程 (如果找到了):

    # 将 1234 替换为你找到的真实 PID
kill -9 1234

方法二：检查 systemd 服务 (如果用 systemctl 安装)

这是在现代 Linux 系统上最标准的管理方式。

  1. 检查服务状态：

    systemctl status nezha-agent

    如果显示 active (running)，说明服务正在运行。

  2. 停止并禁用服务：

    sudo systemctl stop nezha-agent      # 停止当前运行的服务
sudo systemctl disable nezha-agent   # 禁用服务，防止服务器重启后自动运行

方法三：检查 Docker 容器 (如果用 Docker 安装)

  1. 查找 Agent 容器：

    docker ps -a | grep nezha

  2. 停止并删除容器：

    # 将 nezha-agent 替换为你找到的真实容器名或ID
docker stop nezha-agent
docker rm nezha-agent

    如果你是用 docker-compose，请到 docker-compose.yml 文件所在目录执行 docker-compose down。

第二步：删除 Agent 的相关文件

卸载 Agent 包括停止服务、卸载服务，以及删除相关文件。以下是 Ubuntu 系统的卸载步骤：

  1. 停止并卸载服务：

    cd /opt/nezha/agent/
./nezha-agent service uninstall

  2. 删除 Agent 文件夹：

    rm -rf /opt/nezha/agent/

如果安装了多个服务并想要全部卸载，可以使用 Agent 安装脚本的卸载功能：

bash

./agent.sh uninstall
