平台安装失败：'Realtek Ameba Boards (32-bits ARM Cortex-M33 @200MHz):3.1.8' 2 UNKNOWN: Get "https://github.com/ambiot/ambd_arduino/raw/master/Arduino_package/release/ameba_d_toolchain_windows-1.0.1.tar.bz2": read tcp 192.168.71.92:63610->20.205.243.166:443: wsarecv: A connection attempt failed because the connected party did not properly respond after a period of time, or established connection failed because connected host has failed to respond.

原因分析：

  1. Arduino IDE（尤其是 2.x 版）不会自动使用 Windows 系统设置里的“代理”。即便你在 Windows 设置 → 网络和 Internet → 代理 中把“手动设置代理服务器”打开，写了 127.0.0.1:10808（V2Ray 本地 SOCKS5/HTTP 端口），IDE 并不会自动去读取这块设置。
  2. 当你用 Proxifier、Clash 这类“进程级拦截”软件时，如果配置有误，Arduino 进程的网络流量也可能没有被拦截到代理。常见误区包括：
    • 进程名称（Applications）写错了：例如 Proxifier 规则里写成 "arduino ide.exe"，但实际 IDE 进程是 arduino.exe，两者不匹配，拦截不到。
    • DNS 请求没有走代理：默认情况下 Proxifier 是让 Windows 本地先做 DNS 解析，从而把 raw.githubusercontent.com 解析到一个 IP。如果该 IP 无法直连（被屏蔽或限速），就会出现 “no data of the requested type” 或 “lookup … failed” 的错误。
    • 规则优先级排错：如果“默认（Default）”规则比 “Arduino” 规则排在更前面，所有流量都走“Direct”，就根本不会走代理。

Proxifier 是一款在 Windows 和 macOS 平台上常用的网络代理工具，其主要功能是将系统中所有（或指定）应用程序的网络连接“强制”通过一个或多个代理服务器进行转发。

DNS报错

平台安装失败：'Realtek Ameba Boards (32-bits ARM Cortex-M33 @200MHz):3.1.8' 2 UNKNOWN: Get "https://raw.githubusercontent.com/Ameba-AIoT/ameba-arduino-d/master/Arduino_package/release/ameba_d_toolchain_windows-1.0.1.tar.bz2": dial tcp: lookup raw.githubusercontent.com: getaddrinfow: The requested name is valid, but no data of the requested type was found.

DNS 解析依旧本地化，没走到代理服务器

  • 问题表现：错误信息里写到 “lookup raw.githubusercontent.com: getaddrinfow: The requested name is valid, but no data of the requested type was found.” 这实际上是一个 DNS 解析层面的问题：
    • Windows 本地 DNS 未能正确解析 raw.githubusercontent.com，可能解析到了一个被屏蔽或根本没有 A 记录／AAAA 记录的结果。
    • 由于 DNS 解析没有经过代理，IDE 拿到错误的 IP，或者根本没有 IP，就直接连不通。
  • 原因分析：即便你某个代理软件（Clash/Proxifier/V2RayN）已经启动，只要没有设置“DNS 走代理”，默认情况下 DNS 查询还是走本地系统，外网 DNS 解析失败就导致 IDE 进一步发 TCP 请求时直接超时。

双击窗口右下角DNS,取消第一个,勾选第二个

安装完成!还顺便把esp32的包更到最新了,over

⚠️：这个软件不用时需要关掉,否则容易出现ssh远程服务器失败,或者无法上网的情况

记得关掉arduino 首选项里的代理
