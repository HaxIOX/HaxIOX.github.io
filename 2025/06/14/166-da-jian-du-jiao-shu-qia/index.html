安装

官网也太干净了

1. 系统更新与基础软件

sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl unzip

2. 安装和切换到 PHP 7.4

# 安装 PHP7.4 及常用扩展
sudo apt install -y \
  php7.4-cli php7.4-fpm php7.4-mbstring php7.4-xml \
  php7.4-gd php7.4-mysql php7.4-zip php7.4-curl \
  php7.4-bcmath php7.4-fileinfo php-redis

# 将系统默认 php 命令指向 PHP7.4
sudo update-alternatives --set php /usr/bin/php7.4
sudo update-alternatives --set phar /usr/bin/phar7.4
sudo update-alternatives --set phar.phar /usr/bin/phar.phar7.4

# 验证
php -v
sudo systemctl restart php7.4-fpm

3. 安装 Nginx、MySQL、Redis、Supervisor

sudo apt install -y nginx mysql-server redis-server supervisor

4. 配置 MySQL 数据库

  1. 初始化安全设置：

    /udo mysql_secure_installation

  2. 登录 MySQL 并建库建用户（替换 your_password）：

    sudo mysql -u root -p
CREATE DATABASE dujiaoka CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'dujiaoka'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON dujiaoka.* TO 'dujiaoka'@'localhost';
FLUSH PRIVILEGES;
EXIT;

5. 安装 Composer

curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

6. 获取代码并配置环境

cd /var/www
git clone https://github.com/assimon/dujiaoka.git
cd dujiaoka
cp .env.example .env

编辑 .env，设置与你的环境一致的值：

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=dujiaoka
DB_USERNAME=dujiaoka
DB_PASSWORD=your_password

REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379

APP_DEBUG=false

7. 安装 PHP 依赖、生成 key 和迁移

composer install --no-dev -o
php artisan key:generate
php artisan migrate --seed

设置目录权限：

sudo chown -R www-data:www-data storage bootstrap/cache
sudo chmod -R 775 storage bootstrap/cache

清理并缓存配置：

php artisan config:clear
php artisan config:cache

8. 配置 Nginx

在 /etc/nginx/sites-available/dujiaoka 写入：

server {
    listen 80;
    server_name your.domain.com;
    root /var/www/dujiaoka/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php$is_args$query_string;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php7.4-fpm.sock;
    }

    location ~ /\.ht { deny all; }
}

启用并重载：

sudo ln -s /etc/nginx/sites-available/dujiaoka /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

9. 配置 Supervisor 守护队列

创建 /etc/supervisor/conf.d/dujiaoka-worker.conf：

[program:dujiaoka-worker]
command=php /var/www/dujiaoka/artisan queue:work --sleep=3 --tries=3 --daemon
autostart=true
autorestart=true
user=www-data
numprocs=1
redirect_stderr=true
stdout_logfile=/var/www/dujiaoka/storage/logs/worker.log

然后：

sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start dujiaoka-worker:*

或者：

1. 为 artisan 添加可执行权限

sudo chmod +x /var/www/dujiaoka/artisan

这样才能让 Supervisor 或 www-data 用户去调用它。

2. 把 Supervisor 配置写入文件

用你的编辑器（或 tee + heredoc）在 /etc/supervisor/conf.d/dujiaoka-worker.conf 创建以下内容：

sudo tee /etc/supervisor/conf.d/dujiaoka-worker.conf << 'EOF'
[program:dujiaoka-worker]
command=php /var/www/dujiaoka/artisan queue:work --sleep=3 --tries=3 --daemon
autostart=true
autorestart=true
user=www-data
numprocs=1
redirect_stderr=true
stdout_logfile=/var/www/dujiaoka/storage/logs/worker.log
EOF

（如果你更习惯用 nano/vi，也可以 sudo nano /etc/supervisor/conf.d/dujiaoka-worker.conf 然后粘上去、保存。）

3. 重新加载并启动

sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start dujiaoka-worker:*

此时 supervisorctl status 应该能看到 dujiaoka-worker 正在运行。

这样就能避免“命令未找到”和“artisan 权限被拒绝”的问题了。

10. 访问后台

在浏览器中打开：

http://your.domain.com

会自动跳转http://your.domain.com/install,填入设置的数据库名称密码

MySQL 数据库地址：填写 127.0.0.1

MySQL 端口：默认 3306

MySQL 数据库名：你之前建的 名称

MySQL 用户名：用户名

MySQL 密码：你为该用户设置的密码

Redis 连接地址：127.0.0.1

Redis 密码：如果未设置则留空或输入 null

Redis 端口：默认 6379

网站名称：随意填写，比如发卡平台`

网站 URL：https://your.domain.com

后台登录路径：你在 .env 中配置的 ADMIN_PATH，例如 admin

安装后,进入http://your.domain.com/admin

用户名：admin
密码：admin

将.env配置里面的APP_DEBUG设置为false

问题

php7.4无法安装

Ubuntu 22.04 及以后版本官方仓库只提供 PHP 8.1+，要装 PHP 7.4 需要加第三方源。

方案 A：添加 Ondřej Surý 的 PPA，安装 PHP 7.4

  1. 安装添加 PPA 的工具

    sudo apt update
sudo apt install -y software-properties-common

  2. 添加 Ondřej Surý 的 PHP 源

    sudo add-apt-repository ppa:ondrej/php
sudo apt update

  3. 安装 PHP 7.4 及扩展

    sudo apt install -y \
  php7.4-cli php7.4-fpm php7.4-mbstring php7.4-xml \
  php7.4-gd php7.4-mysql php7.4-zip php7.4-curl \
  php7.4-bcmath php7.4-fileinfo php-redis

  4. 切换系统默认 php（可选，但强烈推荐）

    sudo update-alternatives --set php /usr/bin/php7.4
sudo update-alternatives --set phar /usr/bin/phar7.4
sudo update-alternatives --set phar.phar /usr/bin/phar.phar7.4
php -v    # 确认输出为 7.4.x
sudo systemctl restart php7.4-fpm

  5. 回到项目目录，重装依赖并缓存配置

    cd /var/www/dujiaoka
composer install --no-dev -o
php artisan config:clear
php artisan config:cache

这样就能正常生成 vendor/autoload.php 并运行 php artisan 系列命令。

mysql的密码强度

设置为强，则弱密码无法成功创建用户。

  1. 查看当前策略设置

    SHOW VARIABLES LIKE 'validate_password%';

  2. 把策略调到最低

    SET GLOBAL validate_password.policy = LOW;
SET GLOBAL validate_password.length = 4;

  3. 再创建用户

    CREATE USER 'dujiaoka'@'localhost' IDENTIFIED BY 'dujiaoka';
GRANT ALL PRIVILEGES ON dujiaoka.* TO 'dujiaoka'@'localhost';
FLUSH PRIVILEGES;

如果你想彻底卸载这个插件，也可以：

UNINSTALL PLUGIN validate_password;

记得每条 SQL 语句都要以分号 ; 结尾

（禁用默认站点

如果你希望所有访问都落到 dujiaoka 上，也可以直接关掉默认配置：

bash复制编辑sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx

这样不管访问 IP 还是任何其它 “Host”，都会匹配到你 /etc/nginx/sites-available/dujiaoka 这个块。

HTTP 请求已经被 Laravel 强制重定向到 HTTPS

方案一：给站点启用 HTTPS（推荐）

  1. 安装 Certbot

    sudo apt update
sudo apt install -y certbot python3-certbot-nginx

  2. 为你的域名申请并自动配置 SSL

    sudo certbot --nginx -d onecards.tech -d www.onecards.tech
    • 按提示输入邮箱，勾选同意协议。
    • 选 “重定向所有 HTTP 到 HTTPS”

  3. 验证
    申请完成后，Certbot 会自动在 /etc/nginx/sites-enabled/dujiaoka 加入 443 监听配置并重载 Nginx。
    再访问：

    https://onecards.tech/admin

    应该能正确看到 /admin/auth/login 登录页。

  4. 自动续期
    Certbot 会添加定时任务自动续期，你可以执行一次模拟续期验证：

    sudo certbot renew --dry-run

方案二：取消 HTTPS 强制，先用 HTTP 访问

如果短期内不想折腾 SSL，可以临时让 Laravel 不跳转到 HTTPS，回到纯 HTTP：

  1. 编辑 .env：

    APP_URL=http://onecards.tech

  2. 如果有设置“强制 HTTPS”中间件，注释掉或删除 App\Http\Middleware\TrustProxies 中对 \Illuminate\Http\Middleware\EnsureHttps 的调用；或者在 AppServiceProvider 里取消 URL::forceScheme('https')。

  3. 清理缓存并重建：

    php artisan config:clear
php artisan route:clear
php artisan view:clear
php artisan config:cache

  4. 重新加载 Nginx（只要 80 端口在跑，就可以直接访问）：

    sudo systemctl reload nginx

  5. 然后访问：

    http://onecards.tech/admin

    就不会再 302 跳到 HTTPS 了。
