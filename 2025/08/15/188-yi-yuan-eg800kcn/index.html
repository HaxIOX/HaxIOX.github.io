阶段一：前提检查

在开始之前，请务必与服务提供商确认以下事项：

  • 物联网SIM卡: 确认SIM卡已激活，并且您的服务器地址已被添加到该卡的访问白名单中。
  • MQTT服务器: 确认已创建用于设备登录的用户名和密码。
  • 服务器防火墙: 确认已放行MQTT所需端口 (如TCP 1883)。

阶段二：模块网络注册

目标: 让模块连接到移动蜂窝网络并获取IP地址。

  1. 检查SIM卡状态

    AT+CPIN?
    • 预期: +CPIN: READY

  2. 检查信号质量

    AT+CSQ
    • 预期: +CSQ: <rssi>,<ber> (rssi应 > 10, 且不为99)

  3. 检查网络注册

    AT+CGREG?
    • 预期: +CGREG: 0,1 或 +CGREG: 0,5

  4. 配置APN (以ctnet为例)

    AT+CGDCONT=1,"IP","ctnet"
    • 预期: OK

  5. 激活数据通道

    AT+QIACT=1
    • 预期: OK

  6. 确认获取IP

    AT+CGPADDR=1
    • 预期: +CGPADDR: 1,<ip_address>

阶段三：连接MQTT服务器

目标: 建立TCP连接并完成MQTT登录。

  1. (推荐) 配置MQTT版本

    AT+QMTCFG="version",0,4
    • 说明: 设置协议为MQTT v3.1.1，兼容性好。
    • 预期: OK

  2. 建立TCP连接

    AT+QMTOPEN=0,"<服务器域名或IP>",1883
    • 预期: OK 及 +QMTOPEN: 0,0

  3. MQTT登录

    AT+QMTCONN=0,"<客户端ID>","<用户名>","<密码>"
    • 说明: 必须在AT+QMTOPEN成功后立即执行。
    • 预期: OK 及 +QMTCONN: 0,0,0

阶段四：MQTT数据操作

目标: 订阅主题、发布消息、接收消息。

  1. 订阅主题

    AT+QMTSUB=0,1,"topic/sub",1
    • 说明: 订阅主题 “topic/sub”，QoS为1。
    • 预期: OK 及 +QMTSUB: 0,1,0,1

  2. 发布消息

    AT+QMTPUBEX=0,0,0,0,"topic/sub",5
    • 说明: 向 “topic/pub” 主题发布5字节长的消息。
    • 操作: 模块返回 >后，输入消息内容 (例如 hello)。
    • 预期: OK 及 +QMTPUBEX: 0,0,0

  3. 接收消息 (被动)

    • 说明: 当订阅的主题有消息时，模块会自动上报。
    • 格式: +QMTRECV: 0, <msgid>, "<主题>", <长度>, "<消息内容>"

阶段五：断开连接

目标: 干净、优雅地断开连接。

  1. 断开MQTT会话

    AT+QMTDISC=0
    • 预期: OK 及 +QMTDISC: 0,0

  2. 关闭TCP通道

    AT+QMTCLOSE=0
    • 预期: OK 及 +QMTCLOSE: 0,0
