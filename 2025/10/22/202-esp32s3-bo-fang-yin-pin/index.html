ESP32S3播放音频
2025-11-16 19:43:22

原理

数字声音信号 → 电压波形 → 电流驱动 → 空气振动

为什么能还原“人声”

人声主要频率在 300 Hz～3.4 kHz，
如果你每秒取样（采样率）高于两倍这个值（>7 kHz），
理论上就能完整重建原声（奈奎斯特采样定理）。

PWM 的载波频率（60～80 kHz）远高于声音频率，
滤波后得到的平均电压几乎就是原始波形，
再经过功放和扬声器——人声自然还原。

I2S

I²S（Inter-IC Sound） 是一种 数字音频总线标准，用来在芯片之间传输高质量音频数据。它是“芯片到芯片传音频”的专用接口，几乎所有音频设备（如音频DAC、Codec、蓝牙音箱芯片等）都支持它。

一、I²S 是什么

  • 全称：Inter-IC Sound（有时写成 I2S，不是 I²C 的“Sound”版）。
  • 由 Philips（现 NXP）在 1980s 提出。
  • 主要用途：
    👉 把 MCU/SoC（例如 ESP32）输出的 数字音频流（PCM 数据） 传给
    👉 音频解码器、DAC 或数字功放。

²S（Inter-IC Sound） 是一种 数字音频专用通信协议，用于在芯片之间传输音频数据。
它通常需要 3 根信号线（加电源和地），ESP32 / ESP32-S3 都通过这些线输出音频。
音频数据既可以是 WAV（已解码的 PCM），也可以是 MP3（经软件解码后得到的 PCM）。
你不需要再把 WAV 转成数组 —— 可以直接从文件系统（如 SPIFFS、SD 卡）按块读取并发送到 I²S。

简单来说：

I²S 就是音频的“数字线”，替代模拟线路的左右声道线。

二、I²S 信号线说明

I²S 至少包含三根主要信号线（有时加一根 MCLK）：

信号名 中文 作用 对应ESP32引脚示例
BCLK 比特时钟（Bit Clock） 每传一个比特跳一次，控制数据速率 GPIO 5
LRCLK / WS 左右声道时钟（Word Select） 指示当前样本属于左声道或右声道 GPIO 6
DATA / SD 音频数据线 传送 PCM 比特流 GPIO 7
MCLK（可选） 主时钟（Master Clock） 某些高阶 DAC 要求，用于内部PLL同步

三、PCM

PCM 是什么？

PCM 全称是 Pulse Code Modulation（脉冲编码调制）。
它是一种把“模拟声音波形”变成“数字信号（0和1）”的编码方式。

简单说：

PCM 就是“声音数字化”的最基础格式。

二、用类比理解

想象麦克风录到的是一条连续的波形（电压随时间变化）：

模拟声波:   ~~~~~~~~／‾‾‾‾\~~~~~／‾‾‾‾\~~~~

PCM 做的事情就是：

  1. 定时采样 —— 每隔固定时间（比如每秒 16000 次）测量一次波形高度；
  2. 量化编码 —— 把每次测到的电压值转换为一个二进制数字（比如 16-bit 整数）。

于是就得到了一串数字：

PCM 数据:  123, 125, 120, 110, 100, 90, 85, 80, ...

每个数字代表当时声音波的高度（振幅）。
这串数字存进文件，就是最原始的数字音频。

三、PCM 的三个关键参数

参数 含义 示例
采样率（Sample Rate） 每秒采样次数（Hz） 16000 Hz、44100 Hz
量化位数（Bit Depth） 每个采样用多少位存储 8-bit、16-bit、24-bit
声道（Channel） 单声道 / 立体声 Mono / Stereo

例子：

16 kHz、16-bit、单声道 PCM
= 每秒 16000 个采样，每个采样用 2 字节存储。

四、PCM 与 WAV、MP3 的关系

格式 是否压缩 内容 播放方式
PCM（.pcm / 原始流） ❌ 无压缩 纯数字样本 直接送 I²S 播放
WAV ❌ 无压缩 含文件头的 PCM 解析头后输出 PCM
MP3 ✅ 有压缩 压缩的音频帧 解码后得到 PCM 再播放

因此：

  • WAV ≈ “带标签的 PCM”；
  • MP3 需要先“解压缩”成 PCM 才能播放。

I²S 或 DAC 输出的内容就是这些 PCM 数值。
你的程序会：

  1. 从文件系统或网络读音频；
  2. （若是 MP3）先解码成 PCM；
  3. 把 PCM 送到 I²S；
  4. 外部 DAC 或数字功放把这些数字转成模拟电压波形 → 推喇叭。

总结

PCM 是声音的“数字化原型”——
它用一串定时采样的数字（振幅）表示声波，是所有数字音频（WAV、MP3、AAC）的基础。
ESP32-S3 的 I²S 输出的就是 PCM 数据流。

很多人都会疑惑：“PCM只是数字数据，为什么一发出去喇叭就能响？它是不是像PWM那样靠频率在变？”
我们来一步一步拆开讲清楚。

一、PCM 不是靠频率播放的，而是靠数值还原波形

PCM 本质上是：

“每一刻声音波的高度（电压）被记录成一个数字。”

播放时，设备（DAC 或 数字功放）会做反向过程：

  1. 读到一个 PCM 样本，比如 20000；
  2. 立刻输出一个对应电压（比如 0.6V）；
  3. 再过 1/16000 秒（假设采样率16kHz）输出下一个样本对应的电压。

连续快速地这样做，电压就随时间重现了声波的形状，喇叭膜也就随波振动产生声音。

二、PCM “播放”的关键是采样率，而不是PWM频率

概念 含义 举例
采样率 (Sample Rate) 每秒播放多少个PCM点 16,000 Hz（人声）、44,100 Hz（音乐）
PWM频率 每秒PWM方波的震荡次数 一般几十 kHz（40k~100kHz）

区别在于：

  • PCM 是音频的“时间轴”频率（多少次采样代表1秒声音）；
  • PWM 是一种信号调制方式，用方波的占空比代表电压幅度。

所以 PCM 的播放速率 ≠ PWM 的载波频率。
PCM 只是告诉 DAC：“下一刻电压该是多少。”

PCM 是“波形表”，PWM 是“快速闪烁”

PCM（数模转换）

就像画家手动“点描”一条波形：

每点间隔 1/16000 秒，点高低对应声波电压。
DAC 会按这些点连接成平滑波。

PWM（脉宽调制）

就像闪灯一样，用灯亮的时间长短代表电压高低：

亮 80% 的时间 → 表示“电压高”
亮 20% 的时间 → 表示“电压低”
频率很高（比如90kHz）人耳听不出闪烁。

PWM 最后再经过低通滤波（RC电路）才能变回连续电压。
但它带噪、误差大；而 PCM + DAC 直接生成连续模拟电压，音质更好。

播放过程对比（总结）

项目 PWM 方案 PCM + DAC 方案
原理 用高频方波的占空比表示电压 直接输出每个采样点对应的电压
样本格式 通常 8-bit 常用 16-bit 或更高
硬件 只需 GPIO/LEDC 需要 DAC 或 I²S 输出
发热 高（频繁切换） 低（效率高）
音质 差（带宽窄、噪声大） 好（高保真）

总结

PCM 能播放，是因为 每个采样值代表某一时刻声音波的振幅。
播放设备按采样率依次输出这些电压值，就能在喇叭上重现原始声波。
它不是像 PWM 那样靠高频方波，而是靠精准的电压输出节奏（采样率）实现的。

硬件

功率放大器

PAM8302AASCR

扬声器的工作原理：

  1. PAM8302A 输出的交流电流通过线圈（音圈）。
  2. 线圈处在永磁铁的磁场中。
  3. 电流变化 → 磁场变化 → 线圈受到推拉力。
  4. 线圈带动前方的纸盆/振膜前后振动。
  5. 振膜推动空气 → 形成压力波。
  6. 空气波传到你的耳朵 → 你就听到“人声”了。

问题

怪兽声音

采样率 16kHz = 每 62.5 微秒输出一个样本
如果定时不准:
  快了 → 声音变尖 (唐老鸭音)
  慢了 → 声音变粗 (怪兽音)
  
也可能是采样频率设置错误，可以经过测试进行修改

发热

喇叭在进行音频播放时出现发热，大约50度左右，微微烫手，且OUT+-压差未0.2v，但是仅上电没有播放的情况下压差还在，来源于功放芯片

进行下面修改

  • 提高 PWM 载波：40 kHz → 90 kHz
  • 空闲/停止固定 50% 占空（避免直流做功）
  • 全局音量（VOL，默认 0.6）替代大倍数 AUDIO_GAIN
  • 一阶高通去直流（≈20 Hz）
  • 软限幅（±0.8，防削顶）
  • 淡入/淡出（15 ms，避免瞬态大功率）
  • 播放前后 AMP_SD 时序（先回 50% 再关断）

其余逻辑尽可能不动；仍用 8bit PWM

出现杂音问题

1. 为什么您之前的方案会发热和有噪音？

  1. 声音嘶哑/噪音 (GPIO21 -> C45):
    • 您使用的是 PWM 信号 (GPIO21) 来模拟模拟音频。这种方式保真度极低。
    • PWM 信号本身带有很高频率的“载波噪声”，并且其分辨率（量化噪声）远不如真正的 I2S。
    • 这些噪声和失真被模拟功放 (PAM8302) 忠实地放大了，听起来就是“嘶哑”和不清晰的杂音。
  2. 发热严重 (PAM8302):
    • 虽然 PAM8302 本身也是 D 类功放（理论上效率高），但您给它的输入信号是一个高频的 PWM 方波。
    • 模拟功放的输入级可能无法很好地处理这种高频数字信号，导致其工作在非预期的状态（例如，内部开关管可能因输入信号的高频成分而产生无效开关），从而导致效率大幅下降，并将电能转换为了热量。

2. MAX98357 I2S 方案如何解决这些问题

MAX98357 是专门为解决这些问题而设计的。它是一个数字输入 (I2S) D 类放大器 。

方案1：解决噪音和嘶哑问题 (I2S 数字输入)

  • 工作原理： MAX98357 直接接收来自您主控的 I2S 数字音频流（BCLK, LRCLK, DIN 引脚） 。
  • 根本区别： 音频信号在传输过程中全程保持数字格式。模拟转换 (DAC) 是在 MAX98357 芯片内部完成的 。
  • 优势： 这完全避免了 PWM 模拟方案中由信号传输和模拟转换引入的所有噪声、失真和干扰。MAX98357 提供的是行业领先的“Class AB 级”音频性能 ，声音会非常清晰。

方案2：解决高发热问题 (高效率 D 类)

  • 工作原理： MAX98357 是一款专为数字输入优化的高效率 D 类放大器 。
  • 效率： 其效率高达 92% (在 8Ω 负载, 1W 输出时) 。
  • 优势： 极高的效率意味着绝大部分电能都被用于驱动扬声器，只有极少的能量（例如 8%）被浪费为热量。因此，它在正常工作时几乎不会发热，这与您之前“发热严重”的情况会形成鲜明对比。
