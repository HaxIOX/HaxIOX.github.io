Git 指令速查

命令 功能说明
git status 查看工作区和暂存区状态
git diff 查看未暂存的改动
git diff --staged 查看已暂存但未提交的改动
git log 查看提交历史
git branch 列出本地分支
git branch -r 列出远程分支
git checkout <分支名> 切换分支
git checkout -b <新分支> 新建并切换到分支
git merge <分支> 合并指定分支到当前分支

⚠️：多个本地端推送一定要先拉取最新仓库

  1. 切到主干分支：

    git checkout main

  2. 拉取并以变基方式更新到远程最新：

    git pull --rebase origin main

这样本地的 main 就会同步到远程的最新提交。之后再从 main 分出一个 feature 分支去开发，就能保证你的改动是基于最新代码进行的，避免后续合并冲突

执行git pull的时候，相当于以此执行了下面的结合体

  • git fetch ：从云端拉取最新代码
  • git merge：将云端代码与本地代码合并

Clone 远程仓库

  • HTTPS 方式

    git clone https://账户:密码....detector.git

  • SSH 方式

    git clone ..地址

添加、提交与推送

  1. 将修改或新文件加入暂存区

    git add .

    或针对单个文件：

    git add path/to/file.ext

  2. 提交（Commit）

    git commit -m "Add core feature list to README"
    • -m 后面跟的是本次提交的简洁描述。

  3. 关联远程仓库（首次推送时执行，或切换远程时）

    git remote add origin <远程仓库地址>

  4. 推送到远程分支

    git push -u origin main     # 若主分支为 main

    后续只需

    git push

拉取更新与 Rebas

  • 拉取远程最新提交并在本地“衔接”

    git pull origin master --rebase

    --rebase 会把你的本地提交“移到”刚拉下来的远程最新提交之后，避免产生多余的 merge commit。

问题

VIBTEK@DESKTOP-TG673JB MINGW64 ~/Desktop/Detector (master)
$ git push -u origin main
error: src refspec main does not match any
error: failed to push some refs to 'codeup.aliyun.com:67975038cfc95bafa621705d/d
rone/detector.git'

这个错误说明你的远程 master 分支上已经有提交，而你本地的 master 没有包含这些提交，所以 Git 为了防止你不小心覆盖别人工作，拒绝了直接推送。

git fetch origin

git merge origin/master

VIBTEK@DESKTOP-TG673JB MINGW64 ~/Desktop/Detector (master)
$ git merge origin/master
fatal: refusing to merge unrelated histories

当你在本地和远程各自独立 git init 过一次，两个仓库就没有共同的祖先提交，Git 默认会拒绝合并“无关联历史”。

合并两端历史

  1. 拉取远程并允许合并无关联历史

    git pull origin master --allow-unrelated-histories

    或者先 fetch 再 merge：

    git fetch origin
git merge origin/master --allow-unrelated-histories

  2. 处理可能出现的冲突

    • 编辑冲突文件，删除冲突标记（<<<<<<<、=======、>>>>>>>），
    • git add <已解决文件>，
    • 最后 git commit 完成这次合并。

  3. 推送到远程

    git push -u origin master

这样，远程原有的内容和你本地的内容都会被保留并合并到一起。

$ git push origin main
To github.com:god10000/hexo-source.git
! [rejected]            main -> main (non-fast-forward)
error: failed to push some refs to 'github.com:god10000/hexo-source.git'
hint: Updates were rejected because the tip of your current branch is behind
hint: its remote counterpart. If you want to integrate the remote changes,
hint: use 'git pull' before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

  1. 拉取并在本地合并（默认模式）

    git pull origin main

    这会把远程 main 的改动合并到你本地的 main，如果有冲突，Git 会提示你手动解决。

  2. 推送到远程

    git push origin main

如果你不想产生多余的“合并提交”，可以改用 rebase 方式：

git pull --rebase origin main
# 解决冲突后：
git push origin main

方法二：强制推送（不推荐，除非你确定要覆盖远程）

如果你确信远程的改动可以被丢弃，或者远程只有你一个人在用，也可以直接强制推送：

git push --force origin main

⚠️ 警告：强制推送会把远程分支改回你本地的状态，会丢掉远程已有的所有改动。务必在确认没有他人依赖这些提交时才使用。

修改已有远程仓库的地址

  1. 查看当前远程地址

    git remote -v

    这会列出所有远程（如 origin）当前的 fetch 和 push 地址。

  2. 用新的 URL 覆盖旧的 origin

    git remote set-url origin <新仓库地址>

    例如，如果你要改成 SSH 地址：

    git remote set-url origin git@github.com:yourname/new-repo.git

    或者 HTTPS 地址：

    git remote set-url origin https://github.com/yourname/new-repo.git

添加远程 origin

git remote add origin git@github.com:HaxIOX/HaxIOX.github.io.git

 git remote set-url origin git@github.com:HaxIOX/Hexo-source.git

修改历史用户名

源文件

A. 用 git filter-branch

注意：此命令会重写整个历史，所有提交的 hash 都会变，后续需要强推 (--force) 并告知团队重新拉取。

git filter-branch --env-filter '
OLD_EMAIL="old@example.com"
CORRECT_NAME="新用户名"
CORRECT_EMAIL="newemail@example.com"

if [ "$GIT_COMMITTER_EMAIL" = "$OLD_EMAIL" ]
then
    export GIT_COMMITTER_NAME="$CORRECT_NAME"
    export GIT_COMMITTER_EMAIL="$CORRECT_EMAIL"
fi
if [ "$GIT_AUTHOR_EMAIL" = "$OLD_EMAIL" ]
then
    export GIT_AUTHOR_NAME="$CORRECT_NAME"
    export GIT_AUTHOR_EMAIL="$CORRECT_EMAIL"
fi
' --tag-name-filter cat -- --branches --tags

  • 把 OLD_EMAIL、CORRECT_NAME、CORRECT_EMAIL 改成你要替换的内容。

  • 执行完后，清理备份并压缩仓库：

    rm -rf .git/refs/original/
git reflog expire --expire=now --all
git gc --prune=now --aggressive

  • 最后强制推送到远程：

    git push --force --tags origin 'refs/heads/*'

B. 用更现代的 git filter-repo（需要先安装）

git filter-repo \
  --replace-email 'old@example.com=newemail@example.com' \
  --replace-text <(printf 'old-name=>新用户名\n')
git filter-repo --force --commit-callback '
# 把所有 author_name、committer_name 为 OldUser 的提交统一改成 git filter-repo
if commit.author_name == b"wangjiqi":
    commit.author_name = b"HaxIOX"
if commit.committer_name == b"wangjiqi":
    commit.committer_name = b"HaxIOX"
'

具体用法可见官方文档。

或者

git filter-repo \
  --replace-email 旧邮箱=新邮箱 \
  --replace-name 旧用户名=新用户名 \
  --force

强制推送

git push --force --tags origin +refs/heads/*:refs/heads/*
