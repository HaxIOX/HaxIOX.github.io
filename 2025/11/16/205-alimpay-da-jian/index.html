AliMPa 项目搭建笔记

1. 克隆 AliMPay 项目

首先，从 GitHub 上克隆 AliMPay 项目到本地。

git clone https://github.com/ours1505/AliMPay.git
cd AliMPay

2. 配置环境

2.1 安装依赖

AliMPay 项目依赖于 PHP 和 Composer 来安装和管理依赖。

  1. 安装 PHP：

    确保你的系统安装了 PHP，执行以下命令检查 PHP 版本：

    php -v

    如果没有安装 PHP，可以通过以下命令安装：

    sudo apt update
sudo apt install php php-cli php-mbstring php-xml php-zip php-curl php-json

  2. 安装 Composer：

    Composer 是 PHP 的依赖管理工具，用来安装项目的 PHP 依赖。你可以通过以下命令检查是否已安装 Composer：

    composer --version

    如果没有安装，可以通过以下命令安装：

    curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

  3. 安装 PHP 依赖：

    使用 Composer 安装 AliMPay 项目所需的 PHP 依赖：

    composer install

3. 配置 AliMPay

3.1 配置商户信息

在 config 目录下，你会看到一个 codepay.example.json 文件。复制它并重命名为 codepay.json：

cp config/codepay.example.json config/codepay.json

然后编辑 config/codepay.json 文件，填入你从支付宝获取的 商户 ID 和 密钥：

{
  "pid": "你的商户ID",
  "key": "你的商户密钥",
  "notify_url": "http://你的通知地址",
  "return_url": "http://你的返回地址",
  "type": "alipay",
  "sign_type": "MD5"
}
  • pid：你的商户 ID。
  • key：商户密钥（从支付宝提供）。
  • notify_url：支付宝支付成功后的通知 URL。
  • return_url：支付完成后的同步回调 URL。
  • type：支付方式（例如 alipay）。
  • sign_type：签名类型，通常为 MD5。

3.2 配置数据库（可选）

如果你需要使用数据库存储支付数据，可以配置数据库连接。打开 config/database.php 并填写你的数据库配置信息：

<?php

return [
    'host' => 'localhost',
    'port' => '3306',
    'dbname' => 'ali_pay',
    'user' => 'root',
    'password' => 'your_password',
    'charset' => 'utf8'
];

3.3 配置支付宝相关信息

在 config/alipay.php 中，填写支付宝相关信息：

<?php

return [
    'partner'      => '你的商户ID',
    'seller_email' => '你的商户邮箱',
    'private_key'  => '你的商户私钥路径',
    'public_key'   => '支付宝公钥路径',
    'notify_url'   => 'http://你的通知地址',
    'return_url'   => 'http://你的返回地址',
    'sign_type'    => 'RSA2', // 默认RSA2, 如果你的商户没有配置RSA2则使用RSA
];

4. 配置 Apache

4.1 安装 Apache Web 服务器

如果你没有安装 Apache，可以通过以下命令进行安装：

sudo apt update
sudo apt install apache2

4.2 启动 Apache 服务

安装完后，启动 Apache 服务：

sudo systemctl start apache2

4.3 启动 Apache 并设置开机自启

确保 Apache 服务在系统启动时自动启动：

sudo systemctl enable apache2

4.4 检查 Apache 服务状态

确保 Apache 正常运行：

sudo systemctl status apache2

4.5 配置 Apache 虚拟主机

为了让 Apache 指向 AliMPay 项目的目录，我们需要配置虚拟主机。假设你希望将项目部署到 /var/www/html/AliMPay，可以创建一个虚拟主机配置文件。

  1. 创建并编辑虚拟主机配置文件：
sudo nano /etc/apache2/sites-available/alimay.conf
  1. 添加以下内容：
<VirtualHost *:80>
    ServerAdmin webmaster@yourdomain.com
    ServerName yourdomain.com
    DocumentRoot /var/www/html/AliMPay

    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined

    <Directory /var/www/html/AliMPay>
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
  1. 启用配置并重启 Apache：
sudo a2ensite alimay.conf
sudo systemctl restart apache2

4.6 启用 mod_rewrite

如果你需要使用 URL 重写功能（如 .htaccess 配置），确保启用 mod_rewrite：

sudo a2enmod rewrite
sudo systemctl restart apache2

5. 测试支付接口

5.1 配置支付请求

支付请求的代码可以通过 submit.php 或 mapi.php 发起。以下是一个简单的示例：

<?php
require_once 'vendor/autoload.php';

use AliMPay\Core\CodePay;
use AliMPay\Utils\Logger;

// 获取商户信息
$codePay = new CodePay();
$merchantInfo = $codePay->getMerchantInfo();

// 生成支付请求
$params = [
    'pid' => $merchantInfo['pid'],
    'type' => 'alipay',
    'out_trade_no' => '订单号',
    'notify_url' => $merchantInfo['notify_url'],
    'return_url' => $merchantInfo['return_url'],
    'name' => '商品名称',
    'money' => '支付金额',
];

// 签名
$params['sign'] = $codePay->generateSign($params);

// 发起支付请求
$response = $codePay->submitOrder($params);
echo $response;

5.2 支付回调处理

确保你配置的 notify_url 和 return_url 是正确的，并且能够接收支付平台的回调数据。你可以在回调处理函数中处理订单状态的更新。

6. 部署到生产环境

6.1 部署项目

将项目文件上传到生产服务器，或者将其直接复制到生产服务器的 /var/www/html/AliMPay 目录下。

6.2 配置安全性

  1. SSL 配置：确保在生产环境中使用 HTTPS 协议，保护支付数据的安全性。
  2. 环境配置：确保正确设置 config/codepay.json、config/alipay.php 等配置文件，确保商户密钥和支付信息不会暴露。
  3. 日志：启用日志记录功能，监控支付请求和回调，及时发现问题。

6.3 使用 Apache 或 Nginx 部署

你可以使用 Apache 或 Nginx 来作为反向代理或 Web 服务器，托管 AliMPay 项目。

7. 常见问题及排查

7.1 支付请求失败

  • 检查商户 ID 和密钥是否正确。
  • 确保 notify_url 和 return_url 配置正确，并且可以接收到回调数据。
  • 检查接口是否在审核中，未审核通过的接口可能会拒绝请求。

7.2 签名错误

  • 确保签名的生成方法符合支付平台的要求，检查所有参数是否正确拼接，并且密钥没有错误。

7.3 数据库配置问题

如果启用了数据库，确保数据库服务已启动，且连接信息正确。
