199-蓝牙的相关知识
2025-09-29 09:46:44

1. 蓝牙的两种“连接”

  • 经典蓝牙（BR/EDR）
    • 耳机、音箱、手柄这种，大部分都走经典蓝牙。
    • 手机系统蓝牙 UI 原生支持，扫描后直接给你「配对」「连接」。
  • 低功耗蓝牙（BLE，Bluetooth Low Energy）
    • 心率带、手环、智能灯泡、BLE 键盘/鼠标……
    • 基于 GATT Profile（一堆服务/特征值）。
    • 手机系统蓝牙 UI 只会对「它认识的标准 Profile」显示「连接」

👉 也就是说：

  • 经典蓝牙 → 系统 UI 全面支持。
  • BLE → 系统 UI 只支持部分标准 Profile，其他一律不显示。

2. 什么叫“系统 UI 认识的标准 Profile”

蓝牙 SIG（标准组织）定义了一些 标准 GATT 服务（UUID），比如：

  • 0x180D → Heart Rate（心率）
  • 0x180F → Battery Service（电池电量）
  • 0x1812 → HID Service（键盘/鼠标/遥控器）

系统 UI 内置了这些 Profile 的处理逻辑：

  • 如果你的设备广播了 HID Service，手机就会当它是「键盘/鼠标」，UI 里显示「连接」。
  • 如果是 心率计，某些手机系统会直接显示「已连接心率传感器」。
  • 但是如果你广播一个 自定义 Service（比如 0x00FF），手机系统不知道这是干嘛的，就不会在 UI 里给你入口。

3. 为什么 nRF Connect 可以

  • nRF Connect、LightBlue 这类工具 App，直接调用了系统的 BLE API（CoreBluetooth / Android BLE API）。
  • 这些 API 能列出 所有正在广播的设备，无论 UUID 是标准的还是自定义的。
  • 所以在工具 App 里，你可以自由选择「连接」。

👉 换句话说，系统 UI 是挑食的，只吃标准 Profile；工具 App 则是「开发者模式」，什么都能连。

4. 举个例子：你的情况

你现在的 Demo 里是 Battery Service (0x180F)，再加了一个电量特征 (0x2A19)。

  • Android 手机：大多数不会在 UI 里显示「连接」，因为 Android 系统层面电池服务只用来读配件电量，不给用户操作按钮。
  • iOS 手机：更严格，自定义 GATT 一律不在设置里显示「连接」，只能靠 App。

所以结果就是：

  • 在系统蓝牙设置里，你可能只能扫到名字，但点不了「连接」。
  • 在 nRF Connect 里，一切正常，可以连、读写。

5. 解决方法

如果你想在 手机蓝牙设置 UI 里有「连接」按钮：

  1. 让 ESP32 提供 HID Service (0x1812)
    • 手机会把它识别成「键盘/鼠标」，UI 自动出现「连接」按钮。
    • 这是最常见的做法。
  2. 实现其他标准 Profile（比如心率 0x180D）
    • 手机 UI 会把它当成心率带，也能显示「连接」。
    • 但逻辑可能不符合你的应用场景。
  3. 写手机 App
    • 自己用 API 去扫描/连接自定义服务。
    • 但你说了「不想写 App」，所以这条可以忽略。

📌 总结一句话

  • 手机设置里的蓝牙 UI 并不是通用 BLE 调试器，它只对 **标准设备类型（HID、耳机、心率计等）**显示「连接」按钮。
  • 自定义 BLE 服务 在系统 UI 里不会有「连接」，只能在开发工具 App（nRF Connect）里操作，或者写专用 App。
